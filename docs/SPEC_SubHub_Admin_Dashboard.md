## 1. Общее описание

### 1.1 Назначение

SubHub Admin Console — единое веб-приложение для операционного управления сервисом подписок. Является **веб-панелью администратора для мобильного приложения SubHub** (iOS / Android), управляя всем жизненным циклом его пользователей, монетизацией и контентом.

Объединяет пользовательский CRM, финансовый мониторинг, маркетинговые инструменты, аналитику и административные настройки в одном интерфейсе. На текущем этапе бэкенд ещё не реализован; admin-консоль проектируется contract-first и должна работать поверх моковых данных/заглушек до появления единого Python + FastAPI-бэкенда. В будущем этот бэкенд будет обслуживать как мобильное приложение, так и admin-консоль.

### 1.2 Контекст: мобильное приложение

Панель администратора управляет данными, которые конечные пользователи видят в мобильном приложении:

| Мобильное приложение (iOS/Android) | Admin Console |
|-----------------------------------|---------------|
| Экран подписок пользователя | → Управление тарифами и подписками |
| Push-уведомления | → Конструктор рассылок |
| Баллы и достижения | → Геймификация и магазин наград |
| Каталог сервисов (Spotify, Netflix…) | → Топ внешних сервисов, интеграции |
| Профиль и история платежей | → CRM-карточка пользователя |
| Онбординг, промокоды | → Маркетинг, промо-механики |

**Особенности admin-панели в контексте мобильного приложения:**
- Все API-эндпоинты используются совместно с мобильным клиентом — изменения схем данных затрагивают оба клиента
- Push-уведомления отправляются через Firebase (Android) и APNs (iOS) — настраиваются в разделе «Маркетинг»
- Версионирование API (`/api/v1/`) обязательно — мобильные клиенты не обновляются мгновенно
- Churn Prediction обучается на поведенческих данных мобильного приложения (сессии, экраны, время в приложении)

### 1.3 Целевая аудитория

| Роль | Доступные разделы |
|------|------------------|
| Супер-администратор | Все разделы |
| Финансовый менеджер | Дашборд, Подписки, Биллинг |
| Маркетолог | Маркетинг, Геймификация, Аналитика |
| Служба поддержки | Пользователи, Подписки |
| Аналитик | Дашборд, Умная аналитика, Отчёты |

### 1.4 Основные принципы

- **Единое окно:** все операции без переключения между системами
- **Данные в реальном времени:** обновление ключевых метрик каждые 2 минуты
- **Проактивные оповещения:** система сама выявляет аномалии и предупреждает
- **Умная аналитика:** ML-прогнозы, What-if сценарии, A/B тесты

## 2. Технологический стек

### 2.1 Frontend

#### Next.js 16 — App Router

Весь маршрутинг строится на файловой системе App Router. Использование Server Components (RSC) по умолчанию; Client Components подключаются точечно через директиву `"use client"`.

**Соглашения по типам компонентов:**

| Тип | Когда использовать | Примеры в проекте |
|-----|--------------------|------------------|
| Server Component (RSC) | Загрузка данных, нет состояния/событий | Страницы разделов, таблицы с SSR-данными |
| Client Component | Интерактивность, хуки, анимации | Графики, формы, Drawer, модалки |
| Server Action | Мутации данных без API-роутов | Сохранение тарифов, блокировка пользователя |

**Структура директорий:**

```
app/
├── (admin)/                  # Route Group — общий layout с сайдбаром
│   ├── layout.tsx            # AdminShell: sidebar + header + toast-provider
│   ├── dashboard/
│   │   └── page.tsx          # RSC — загружает KPI, аномалии, топ-сервисы
│   ├── users/
│   │   ├── page.tsx          # RSC — таблица пользователей (серверная пагинация)
│   │   └── [id]/
│   │       └── page.tsx      # RSC — карточка пользователя
│   ├── subscriptions/
│   ├── billing/
│   ├── marketing/
│   ├── gamification/
│   ├── analytics/
│   ├── smart-analytics/

├── api/                      # Route Handlers (webhooks, SSE)
```

**Server Actions — соглашения:**

```typescript
// actions/users.ts
"use server";

import { z } from "zod";
import { revalidatePath } from "next/cache";

const BlockUserSchema = z.object({ userId: z.string().uuid(), reason: z.string().min(1) });

export async function blockUser(formData: FormData) {
  const parsed = BlockUserSchema.safeParse(Object.fromEntries(formData));
  if (!parsed.success) return { error: parsed.error.flatten() };

  await db.user.update({ where: { id: parsed.data.userId }, data: { status: "BLOCKED" } });
  revalidatePath("/users");
  return { success: true };
}
```

**Стратегии кэширования данных:**

| Тип данных | Стратегия | Ревалидация |
|-----------|-----------|-------------|
| KPI метрики | `fetch(..., { next: { revalidate: 120 } })` | Каждые 2 мин |
| Список пользователей | `cache: "no-store"` | Всегда свежие |
| Тарифные планы | `revalidateTag("tariffs")` | По мутации |
| ML-прогнозы | `{ next: { revalidate: 86400 } }` | Раз в сутки |
| Настройки | `cache: "force-cache"` + `revalidateTag` | По сохранению |

#### HeroUI 3

Основа для большинства UI-компонентов. Построен поверх **React Aria Components** + Tailwind CSS v4. 

> **HeroUI v3 полностью убрал Framer Motion из зависимостей.** Анимации компонентов теперь реализованы через нативные CSS-переходы и `@keyframes`. Это даёт меньший бандл и GPU-ускорение из коробки. #### motion (бывший Framer Motion)

Пакет `framer-motion` переименован в **`motion`** начиная с v11. Актуальный импорт: `from "motion/react"`.

Используется для страничных переходов, появления карточек и интерактивных микро-анимаций. **Не используется внутри компонентов HeroUI** — у них собственные CSS-анимации.

**Варианты анимаций (переиспользуемые):**

```typescript
// lib/motion-variants.ts
import { Variants } from "motion/react";

export const fadeInUp = {
  hidden:  { opacity: 0, y: 16 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.3, ease: [0.4, 0, 0.2, 1] } },
};

export const staggerChildren = {
  visible: { transition: { staggerChildren: 0.07 } },
};

export const scaleIn = {
  hidden:  { opacity: 0, scale: 0.95 },
  visible: { opacity: 1, scale: 1, transition: { duration: 0.25, ease: [0.34, 1.56, 0.64, 1] } },
};

export const slideInRight = {
  hidden:  { opacity: 0, x: 32 },
  visible: { opacity: 1, x: 0,  transition: { duration: 0.3, ease: [0.4, 0, 0.2, 1] } },
};
```

**Применение по компонентам:**

| Компонент | Анимация | Описание |
|-----------|----------|----------|
| KPI-карточки | `staggerChildren` + `fadeInUp` | Появление по очереди при загрузке страницы |
| Drawer (боковая панель) | `slideInRight` | Выезд справа |
| Модальные окна | `scaleIn` | Масштабирование из центра |
| Тостеры | `slideInRight` | Влетают справа |
| Числа в KPI (counter) | `useSpring` + `useTransform` | Плавный счётчик при изменении значения |
| Аномалии (новая) | `fadeInUp` | Появление при получении через SSE |
| Страничные переходы | `AnimatePresence` + `fadeInUp` | Между разделами |

**Пример KPI-карточек с анимацией:**

```typescript
"use client";
import { motion } from "motion/react";
import { staggerChildren, fadeInUp } from "@/lib/motion-variants";

export function KpiGrid({ kpis }: { kpis: KpiData[] }) {
  return (
    <motion.div
      className="grid grid-cols-4 gap-4"
      variants={staggerChildren}
      initial="hidden"
      animate="visible"
    >
      {kpis.map((kpi) => (
        <motion.div key={kpi.id} variants={fadeInUp}>
          <KpiCard {...kpi} />
        </motion.div>
      ))}
    </motion.div>
  );
}
```

#### GSAP

Используется для сложных, высокопроизводительных анимаций, где Framer Motion недостаточен.

**Сценарии использования:**

| Компонент | GSAP-техника | Описание |
|-----------|-------------|----------|
| Кольцевая диаграмма «Риск оттока» | `gsap.to(dashOffset)` | Плавное заполнение stroke-dashoffset при загрузке |
| Числовой счётчик KPI | `gsap.to(obj, { val })` | Анимация числа от 0 до значения |
| Линия графика MRR | `DrawSVGPlugin` | Отрисовка линии слева направо |
| Прогресс-бары сервисов | `gsap.fromTo(width)` | Заполнение баров при появлении |
| Страница аналитики | `ScrollTrigger` | Секции появляются при скролле |

**Пример анимации числового счётчика:**

```typescript
"use client";
import { useEffect, useRef } from "react";
import gsap from "gsap";

export function AnimatedCounter({ value, prefix = "", suffix = "" }: CounterProps) {
  const ref = useRef<HTMLSpanElement>(null);

  useEffect(() => {
    const ctx = gsap.context(() => {
      gsap.fromTo(
        { val: 0 },
        { val: value, duration: 1.2, ease: "power2.out",
          onUpdate() { if (ref.current) ref.current.textContent = `${prefix}${Math.round(this.targets()[0].val).toLocaleString("ru-RU")}${suffix}`; }
        }
      );
    });
    return () => ctx.revert();
  }, [value]);

  return <span ref={ref}>{prefix}0{suffix}</span>;
}
```

> **Правило:** `motion` — для layout-анимаций и страничных переходов. GSAP — для SVG-анимаций и числовых счётчиков. HeroUI-компоненты анимируются встроенным CSS — не оборачивать их в `<motion.*>` без необходимости.

### 2.3 Формы и валидация

#### React Hook Form + Zod

Все формы строятся через `useForm` с резолвером Zod. Server Actions принимают уже провалидированные данные.

**Паттерн формы:**

```typescript
// schemas/tariff.schema.ts
import { z } from "zod"; // v4 теперь по умолчанию

export const TariffSchema = z.object({
  name:          z.string().min(1, "Обязательное поле").max(80),
  price:         z.coerce.number().positive("Цена должна быть > 0"),
  currency:      z.enum(["RUB", "USD", "EUR"]),
  period:        z.enum(["month", "year", "once"]),
  trialDays:     z.coerce.number().int().min(0).max(90).default(0),
  serviceIds:    z.array(z.string().uuid()).min(1, "Выберите хотя бы один сервис"),
  isPublic:      z.boolean().default(true),
  description:   z.string().max(500).optional(),
});

export type TariffFormData = z.infer<typeof TariffSchema>;
```

```typescript
// components/forms/TariffForm.tsx
"use client";
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { TariffSchema, type TariffFormData } from "@/schemas/tariff.schema";
import { createTariff } from "@/actions/tariffs";

export function TariffForm() {
  const { register, handleSubmit, formState: { errors, isSubmitting } } = useForm<TariffFormData>({
    resolver: zodResolver(TariffSchema),
    defaultValues: { currency: "RUB", period: "month", trialDays: 0, isPublic: true },
  });

  const onSubmit = async (data: TariffFormData) => {
    const result = await createTariff(data);
    if (result.error) { /* обработка */ }
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {/* поля */}
    </form>
  );
}
```

**Стандартные схемы валидации:**

| Форма | Ключевые правила |
|-------|----------------|
| Тариф | price > 0, serviceIds.length ≥ 1, trialDays 0–90 |
| Промокод | code: regex `/^[A-Z0-9_-]{3,20}$/`, discount > 0, endDate > startDate |
| Рассылка | subject ≥ 5 символов, segmentId uuid, scheduledAt > now (если отложенная) |
| Блокировка пользователя | reason ≥ 10 символов |
| Автоматизация | triggers.length ≥ 1, actions.length ≥ 1 |
| What-If параметры | priceChange -50…+100, churnDelta -5…0, horizon 1…24 |

**Отображение ошибок:**

```typescript
// Единый компонент ошибки поля
import { motion } from "motion/react";
export function FieldError({ message }: { message?: string }) {
  if (!message) return null;
  return (
    <motion.p
      className="text-danger text-xs mt-1"
      initial={{ opacity: 0, y: -4 }}
      animate={{ opacity: 1, y: 0 }}
    >
      {message}
    </motion.p>
  );
}
```

- Прогноз LTV
- ROI изменения
- График: текущий сценарий vs прогнозируемый

### 8.2 A/B Тесты

**Конструктор теста:**

| Шаг | Описание |
|-----|----------|
| 1. Название | Текстовое поле |
| 2. Гипотеза | Текстовое поле |
| 3. Тип | Цена / Лендинг / Email-тема / Онбординг |
| 4. Варианты | Вариант A (контроль) + Вариант B (+C опционально) |
| 5. Метрика | Конверсия / Churn / ARPU / Открываемость |
| 6. Аудитория | % разбивка (50/50 или кастом) + сегмент |
| 7. Длительность | Дата начала / конца или авто-стоп по significance |

**Dashboard теста:**
- Статус: Черновик / Запущен / Остановлен / Завершён / Победитель определён
- Метрики в реальном времени по вариантам
- Статистическая значимость (p-value, confidence interval)
- Кнопка «Применить победителя»

### 8.3 ML-модели и прогнозы

#### Churn Prediction

| Элемент | Описание |
|---------|----------|
| Модель | Gradient Boosting / Random Forest (конфигурируется) |
| Обновление | Ежедневно в 03:00 |
| Выходные данные | Вероятность оттока для каждого пользователя (0–1) |
| Сегменты риска | Высокий (>0.7), Средний (0.4–0.7), Низкий (<0.4) |
| Факторы | Топ-10 факторов влияния (feature importance) |
| Действие | «Создать сегмент из группы риска» → автоматически в CRM |

#### LTV Prediction

- Прогноз LTV для каждого пользователя на 6/12/24 месяца
- Сегментация: Champions, Loyal, At Risk, Lost
- Тепловая карта когорт

#### Прогноз MRR

- Экспоненциальное сглаживание + ARIMA
- Доверительный интервал 80% / 95%
- Сравнение с целевыми показателями

### 8.4 Конструктор автоматизаций

**Триггеры:**

| Тип | Примеры |
|-----|---------|
| Временной | «Пользователь не заходил N дней» |
| Событийный | «Подписка истекает через N дней» |
| Метрический | «Баллы опустились ниже N» |
| Поведенческий | «Пользователь просмотрел раздел отмены» |

**Условия (AND/OR конструктор):**
- Тариф = X
- Страна = X
- Риск оттока > High
- Количество транзакций < N

**Действия:**
- Отправить email из шаблона
- Отправить push-уведомление
- Начислить N баллов
- Присвоить тег
- Добавить в A/B тест
- Вызвать webhook

**Пример сценария:**
```
ЕСЛИ пользователь не заходил 14 дней
  И подписка активна
  И риск оттока = Высокий
ТОГДА:
  → Отправить email «Мы скучаем»
  → Подождать 3 дня
  → ЕСЛИ не вошёл → Начислить 100 бонусных баллов
  → ЕСЛИ вошёл → Отметить как «Возвращённый»
```

**Управление сценариями:**
- Список с фильтрацией по статусу
- Статистика срабатываний
- Вкл/Выкл без удаления
- Версионирование

## 14. Обработка ошибок на фронтенде

### 14.1 Коды ошибок от бэкенда

| HTTP | Код | Описание |
|------|-----|----------|
| 400 | `VALIDATION_ERROR` | Некорректные данные запроса |
| 400 | `INVALID_TRANSITION` | Недопустимый переход статуса |
| 401 | `UNAUTHORIZED` | Токен отсутствует или невалиден |
| 401 | `TOKEN_EXPIRED` | JWT истёк — нужен silent refresh |
| 403 | `FORBIDDEN` | Недостаточно прав |
| 404 | `USER_NOT_FOUND` | Пользователь не найден |
| 404 | `SUBSCRIPTION_NOT_FOUND` | Подписка не найдена |
| 404 | `TARIFF_NOT_FOUND` | Тариф не найден |
| 409 | `EMAIL_ALREADY_EXISTS` | Email занят |
| 409 | `SUBSCRIPTION_ALREADY_ACTIVE` | Подписка уже активна |
| 422 | `UNPROCESSABLE_ENTITY` | Ошибка валидации Pydantic — `details` содержит ошибки по полям |
| 429 | `RATE_LIMIT_EXCEEDED` | Превышен лимит запросов |
| 500 | `INTERNAL_ERROR` | Внутренняя ошибка сервера |
| 503 | `SERVICE_UNAVAILABLE` | Зависимый сервис недоступен |

### 14.2 Глобальный обработчик ошибок (React Query)

```typescript
// lib/query-client.ts
import { QueryClient } from "@tanstack/react-query";
import { toast } from "@heroui/react";

export const queryClient = new QueryClient({
  defaultOptions: {
    mutations: {
      onError: (error: unknown) => {
        const err = error as ApiError;
        switch (err.code) {
          case "TOKEN_EXPIRED":
            // обрабатывается axios-интерцептором — не показываем toast
            break;
          case "UNAUTHORIZED":
            window.location.href = "/login";
            break;
          case "FORBIDDEN":
            toast.error("Недостаточно прав для этого действия");
            break;
          case "RATE_LIMIT_EXCEEDED":
            toast.error("Слишком много запросов. Попробуйте через 30 секунд");
            break;
          case "INTERNAL_ERROR":
            toast.error(`Что-то пошло не так (ID: ${err.request_id})`);
            break;
          default:
            toast.error(err.message ?? "Неизвестная ошибка");
        }
      },
    },
  },
});
```

### 14.3 Axios-интерцептор: Silent Token Refresh

```typescript
// lib/api.ts
import axios from "axios";

const api = axios.create({ baseURL: "/api/v1", withCredentials: true });

let isRefreshing = false;
let failedQueue: Array<{ resolve: Function; reject: Function }> = [];

api.interceptors.response.use(
  (res) => res,
  async (error) => {
    const original = error.config;
    const code = error.response?.data?.error?.code;

    if (code === "TOKEN_EXPIRED" && !original._retry) {
      if (isRefreshing) {
        return new Promise((resolve, reject) =>
          failedQueue.push({ resolve, reject })
        ).then(() => api(original));
      }

      original._retry = true;
      isRefreshing = true;

      try {
        await api.post("/auth/refresh");
        failedQueue.forEach(({ resolve }) => resolve());
        return api(original);
      } catch {
        failedQueue.forEach(({ reject }) => reject());
        window.location.href = "/login";
      } finally {
        isRefreshing = false;
        failedQueue = [];
      }
    }
    return Promise.reject(error);
  }
);

export default api;
```

### 14.4 Обработка ошибок в формах (422)

При получении `422 UNPROCESSABLE_ENTITY` поле `details` содержит объект вида:
```json
{ "price": ["Значение должно быть больше 0"], "service_ids": ["Выберите хотя бы один сервис"] }
```

Фронтенд проставляет ошибки в React Hook Form через `setError`:

```typescript
const onSubmit = async (data: TariffFormData) => {
  try {
    await createTariff(data);
    toast.success("Тариф создан");
  } catch (error) {
    const err = error as ApiError;
    if (err.code === "UNPROCESSABLE_ENTITY" && err.details) {
      Object.entries(err.details).forEach(([field, messages]) => {
        setError(field as keyof TariffFormData, {
          type: "server",
          message: messages[0],
        });
      });
    }
  }
};
```

### 14.5 Обработка ошибок в Server Actions (Next.js)

```typescript
// app/(admin)/users/[id]/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export type ActionResult =
  | { success: true }
  | { success: false; code: string; message: string; details?: Record<string, string[]> };

export async function blockUserAction(userId: string, reason: string): Promise<ActionResult> {
  try {
    await api.post(`/users/${userId}/block`, { reason });
    revalidatePath(`/users/${userId}`);
    return { success: true };
  } catch (error) {
    const err = error as ApiError;
    return {
      success: false,
      code:    err.code,
      message: err.message,
      details: err.details,
    };
  }
}
```

```typescript
// Использование в Client Component
const result = await blockUserAction(userId, reason);
if (!result.success) {
  toast.error(result.message);
}
```

### 14.6 Матрица: ошибка → поведение UI

| HTTP | Код | Фронтенд-реакция |
|------|-----|-----------------|
| 401 `TOKEN_EXPIRED` | Silent refresh → повтор запроса | Пользователь ничего не видит |
| 401 `UNAUTHORIZED` | Redirect → `/login` | — |
| 403 `FORBIDDEN` | Toast «Недостаточно прав» | Кнопка блокируется |
| 404 `*_NOT_FOUND` | Toast «Не найдено» + закрыть Drawer | — |
| 409 `CONFLICT` | Toast с текстом `message` | — |
| 422 `UNPROCESSABLE_ENTITY` | `setError()` на конкретные поля | Без toast |
| 429 `RATE_LIMIT_EXCEEDED` | Toast + блокировка кнопки на 30 сек | — |
| 500 `INTERNAL_ERROR` | Toast «Что-то пошло не так (ID: {request_id})» | ID для поддержки |
| 503 `SERVICE_UNAVAILABLE` | Full-page banner «Сервис временно недоступен» | — |
